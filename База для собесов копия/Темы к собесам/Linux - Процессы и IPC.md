#### **Как убить процесс в Linux?**

Чтобы завершить процесс в Linux, необходимо узнать его PID (Process ID) и выполнить команду `kill`:

- **kill -15 [PID]** — отправляет сигнал `SIGTERM`, который можно перехватить. Дает процессу время корректно завершиться.
    
- **kill -9 [PID]** — отправляет `SIGKILL`, завершает процесс немедленно(будто отключаем [[obsidian/База для собесов копия/Темы к собесам/Компьютер|компьютер]] от розетки), без возможности обработки.
    

#### **Основные команды управления файлами и процессами**

- `ps` — отображение списка процессов.
    
- `ls` — просмотр содержимого каталога.
    
- `cd` — смена текущего каталога.
    
- `mkdir` — создание нового каталога.
    
- `touch` — создание нового файла.
    
- `cp` — копирование файлов.
    
- `rm` — удаление файлов.
    
- `cat` — просмотр содержимого файла.
    
- `sudo` — выполнение команд от имени суперпользователя.
    
- `grep` — поиск в файлах.
    
- `echo` — запись информации в файл.
    
- `clear` — очистка терминала.

## **Межпроцессное взаимодействие (IPC)**

IPC (**Inter-Process Communication**) — это механизм обмена данными между процессами. Он реализуется через:

1. **Обмен сообщениями**
    
2. **Синхронизацию**
    
3. **Разделяемую память**
    
4. **Удаленные вызовы процедур (RPC)**

#### **Сигнал (Signal)**

Сигнал — это асинхронное уведомление процесса о событии. Например, `kill` может отправить сигнал завершения.

#### **Сокет (Socket)**

Сокет — это объект, представляющий конечную точку сетевого соединения.

- **Клиентские сокеты** — используются в клиентских приложениях.
    
- **Серверные сокеты** — используются на веб-серверах.
    

#### **Канал (Pipe)**

Каналы используются для перенаправления вывода одного процесса на вход другого (`|`).

#### **Разделяемая память (Shared Memory)**

Наиболее быстрый способ обмена данными между процессами. Позволяет передавать информацию без переключения контекста в ядре.

## **Файловые дескрипторы**

Файловый дескриптор — это идентификатор потока ввода-вывода. Он может быть связан с:

- **Файлом**
    
- **Каталогом**
    
- **Сокетом**
    

При открытии файла ОС создает запись в системной таблице дескрипторов.

![[Untitled 2 3.png]]

**Стандартные дескрипторы:**

- `0` — стандартный ввод (`stdin`).
    
- `1` — стандартный вывод (`stdout`).
    
- `2` — стандартный поток ошибок (`stderr`).
    

#### **Проблемы с файловыми дескрипторами**

Ошибка `Bad file descriptor` возникает, когда один поток закрывает файл, а другой пытается к нему обратиться.

## **Зомби-процессы**

Зомби-процесс — это процесс, завершивший выполнение, но все еще присутствующий в системе, пока родительский процесс не получит код завершения через `wait()`.

- Зомби-процессы **не потребляют память**, но **блокируют записи в таблице процессов**.
    
- Они **не принимают сигналы** и **не могут быть завершены** `**kill**`.

## **Создание процессов в Linux**

#### **fork() — механизм копирования процессов**

Вызов `fork()` создает новый процесс (дочерний), который является копией родительского.

#### **pipe — механизм межпроцессного взаимодействия**

Pipe создает канал передачи данных между двумя процессами:

```
P1 (отправляет) ---> Pipe ---> P2 (читает)
```

Используется для передачи байтовых данных между процессами.
