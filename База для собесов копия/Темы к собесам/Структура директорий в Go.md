## **Основные директории**

### **/cmd**

Главная точка входа в приложение. Здесь находятся исполняемые файлы для разных сервисов проекта.

### **/internal**

Внутренний код, который **не может** быть импортирован из других проектов. Здесь хранится бизнес-логика и работа с БД.

### **/pkg**

Библиотеки, которые могут использоваться в других проектах. Здесь можно хранить кастомные или общие модули, которые повторно используются.

> **Важно:** если проект небольшой, можно не использовать `/pkg`, а хранить код в `/internal`.

## **Многослойная [[obsidian/База для собесов копия/Темы к собесам/Архитектура программных систем|архитектура]]**

_==_**_Транспортный слой:_**_==_

Сетевой уровень приложения, на котором конечный пользователь взаимодействует с системой. После обработки запроса данные передаются **на слой ниже**.

- Обрабатывает HTTP-запросы, gRPC, WebSockets
    
- Передаёт данные в бизнес-слой
    
- Возвращает обработанный ответ клиенту
    

_==_**_Бизнес-слой:_**_==_

Содержит основную бизнес-логику приложения. Если требуется работа с БД, **обращается на слой базы данных**.

- Операции с пользователями, платежами, заказами и др.
    
- Проверка входных данных, валидация
    
- Запросы к базам данных через DAO/Repository
    

_==_**_Слой базы данных:_**_==_

Отвечает за взаимодействие с постоянным хранилищем (БД, кеш, файловая система).

- Запись, чтение, обновление данных
    
- Транзакции, индексы, кеширование
    
- ORM или работа через `sql.DB`

## **Конфигурации и документация**

### **/configs**

- Статические конфигурации приложения, например, **YAML-файлы**.

### **/api**

- Документация API (OpenAPI, Swagger, JSON Schema, файлы определения протоколов).

### **/build**

- Файлы сборки: конфигурации Docker, CI/CD, Makefile.

### **/deployments**

- Файлы для развертывания: **Kubernetes-манифесты, Docker Compose, Helm-диаграммы**.

### **/docs**

- Документация по коду и архитектуре. Может дополнять **Godoc**.

### **README.md**

- Описание проекта: цели, установка, зависимости, примеры использования.

## **Дополнительные директории**

### **/scripts**

- Скрипты для сборки, тестирования, деплоя. Позволяют упростить Makefile.

### **/testdata**

- Тестовые данные и внешние ресурсы для тестирования.

### **/tools**

- Скрипты и утилиты, поддерживающие работу проекта. Они могут импортировать код из `/pkg` и `/internal`.

### **/assets**

- Статические ресурсы: изображения, шрифты, иконки.

### **/web**

- Для веб-приложений: статические файлы, шаблоны, клиентская часть.

### **/migrations**

- Файлы миграций для работы с базами данных (например, `.sql`-скрипты).
