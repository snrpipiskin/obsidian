## **Основные возможности Kubernetes**

Kubernetes позволяет **автоматизировать**:

- Управление контейнерами
    
- Мониторинг
    
- Развертывание
    
- Масштабирование

---

## **Архитектура Kubernetes**

Контейнеры **запускаются внутри Подов (Pod)** — это минимальная единица развертывания.

![[Untitled 1 3.png]]

- **POD** — это обертка над контейнером.
    
- Если нагрузка увеличивается, создаются **новые поды**.
    
- Поды **пересоздаются** при сбоях.
    
- Каждый под имеет **свой IP-адрес** и может взаимодействовать с другими подами.
    

### **Сервис Kubernetes**

- Объединяет поды в группы.
    
- Выдает группе подов **постоянный IP**.
    
- Обеспечивает **балансировку нагрузки** между подами.
    

---

## **Настройка Kubernetes**

1. Отправляем **запрос на API-сервер** (на **мастер-ноде**).
    
2. Передаем **файл конфигурации YAML**.
    
3. Описываем в **декларативном стиле** (что нужно получить, а не как).
    

### **Контроллер-менеджер**

- Следит за состоянием подов.
    
- Автоматически **перезапускает поды**, если они упали.
    

---

## **Динамическое управление приложениями**

- Запущенное приложение **всегда в движении**.
    
- Kubernetes **заменяет упавшие сервисы** на новые.
    

### **Контейнеры в Kubernetes**

Контейнер — это **изолированный, легковесный и переносимый пакет** с ПО, который содержит **все необходимые зависимости** для работы в любой среде.

Kubernetes позволяет:

- Разворачивать контейнеры **на разных машинах**.
    
- Автоматически **перезапускать контейнеры** при сбоях.
    
- **Гибко масштабировать** инфраструктуру по нагрузке.
  

**Сервис:**  
- логически объединяет поды в группы, стоит между ними  
- выдает группе Подов постоянный IP  
- обеспечивает Балансировку нагрузки  

  

**Настройка Кубера:**  
- Запрос на API сервер - на Мастере  
- Передаем файл конфигурации YAML  
- Описываем в Декларативном стиле  
(То что хотим получить, машина сделает за нас)  

  

**Контроллер менеджер:**  
- Поднимает наши Поды, если они упали
