# **Git – управление версиями**

Git — это распределенная система контроля версий, которая позволяет отслеживать изменения в коде, управлять ветками, выполнять слияния и восстановление предыдущих состояний проекта.

---

## **Git Merge vs. Git Rebase**

![[Untitled 2.png]]

### **Git Merge** (Слияние)

Слияние принимает содержимое одной ветки и объединяет его с целевой веткой.  
При этом создается новый коммит слияния, который фиксирует факт объединения.

- **Плюсы:**
    
    - История изменений сохраняется в первозданном виде.
    - Видно, какие ветки участвовали в разработке.
- **Минусы:**
    
    - В истории появляются дополнительные коммиты merge, что может усложнить ревизию изменений.

---

### **Git Rebase** (Перебазирование)

Rebase сжимает все изменения в один «патч» и затем интегрирует его в целевую ветку.  
В отличие от слияния, Rebase **перезаписывает историю**, создавая новые коммиты на основе обновленной ветки.

- **Плюсы:**
    
    - Поддерживается чистая и линейная история.
    - Упрощает ревизию изменений.
- **Минусы:**
    
    - **Меняет историю**, что может привести к проблемам при совместной разработке.
    - Не рекомендуется использовать в общих ветках.

Используйте Rebase **только** в следующих случаях:

1. Когда работаете **локально** и не делились кодом с другими.
2. Когда **готовите код к ревью** и хотите объединить коммиты.
3. Когда ветка готова к **интеграции в основную ветку**.

## **Основные команды Git**

### **Настройка пользователя**

```sh
git config --global user.name "Ваше имя"
git config --global user.email "Ваш email"
```

### **Создание репозитория**

```sh
git init
```

Инициализирует новый репозиторий.

### **Добавление файлов в коммит**

```sh
git add .
git commit -m "Описание изменений"
```

### **Просмотр истории коммитов**

```sh
git log
git log --oneline
git log --graph --decorate --all
```

### **Просмотр изменений**

```sh
git status
git diff
git show <коммит>
```

### **Работа с ветками**

Создание новой ветки:

```sh
git branch <название_ветки>
```

Переключение на ветку:

```sh
git checkout <название_ветки>
```

Создание и переключение:

```sh
git checkout -b <название_ветки>
```

Удаление ветки:

```sh
git branch -d <название_ветки>
```

### **Слияние веток**

```sh
git merge <название_ветки>
```

Прервать слияние (если возник конфликт):

```sh
git merge --abort
```

### **Перебазирование (Rebase)**

```sh
git rebase <название_ветки>
```

## **Работа с удаленными репозиториями**

### **Подключение удаленного репозитория**

```sh
git remote add origin <URL-репозитория>
```

### **Отправка изменений в удаленный репозиторий**

```sh
git push origin <ветка>
```

### **Получение изменений из удаленного репозитория**

```sh
git pull origin <ветка>
```

### **Обновление локальной ветки без merge-коммита**

```sh
git pull --rebase origin <ветка>
```

---

## **Восстановление изменений**

### **Отмена последнего коммита**

```sh
git reset --soft HEAD~1
```

Сохраняет изменения, но убирает коммит.

```sh
git reset --hard HEAD~1
```

Полностью удаляет изменения.

### **Откат конкретного коммита**

```sh
git revert <хеш_коммита>
```

Создает новый коммит, отменяющий изменения указанного коммита.

### **Переключение на более раннее состояние**

```sh
git checkout <хеш_коммита>
```

Позволяет временно переключиться на конкретную версию проекта.

## **Git и командная работа**

- **Перед началом работы:**
    
    ```sh
    git pull origin main
    ```
    
    Обновите локальный репозиторий, чтобы избежать конфликтов.
    
- **Создайте отдельную ветку для задачи:**
    
    ```sh
    git checkout -b feature/my-task
    ```
    
- **Вносите изменения и коммитьте:**
    
    ```sh
    git add .
    git commit -m "Добавил новую фичу"
    ```
    
- **Периодически синхронизируйтесь с основным репозиторием:**
    
    ```sh
    git pull --rebase origin main
    ```
    
- **Когда фича готова, отправьте её в репозиторий:**
    
    ```sh
    git push origin feature/my-task
    ```
    
- **Создайте Pull Request и дождитесь ревью.**
