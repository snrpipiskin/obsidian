Когда стоит вызывать runtime.GC() вручную?

---

**Почти никогда в production коде.** Go runtime сам знает, когда запускать GC эффективнее.

**Допустимые случаи:**
- **Бенчмарки**: Очистить состояние перед тестом для чистоты измерений
- **Batch-обработка**: Между батчами при обработке больших объёмов данных
- **Тестирование**: При тестировании поведения самого GC

**Почему не стоит использовать в production:**
- Runtime лучше знает оптимальное время запуска
- Ручной вызов может нарушить эффективность работы GC
- Может привести к лишним паузам в неподходящее время
