```go
package main

type Foo struct{}

func (f *Foo) A() {}
func (f *Foo) B() {}
func (f *Foo) C() {}

type AB interface {
	A()
	B()
}

type BC interface {
	B()
	C()
}

func main() {
	var f AB = &Foo{}
	y := f.(BC) // сработает ли такой type-assertion?
	y.A()       // а этот вызов?
	_ = y
}
```

---

**Ответ:** Программа **не скомпилируется** (ошибка `y.A undefined`).

**Почему:**
1. `y := f.(BC)` — отработает успешно. Тип под интерфейсом (`*Foo`) реализует `BC` (есть методы `B()` и `C()`). 
2. `y.A()` — вызовет ошибку компиляции. У переменной `y` тип интерфейса `BC`, который "не знает" о методе `A()`, поэтому вызвать его нельзя. Компилятору неважно, что `*Foo` физически имеет этот метод.
