Что показывает EXPLAIN ANALYZE?

---

`EXPLAIN ANALYZE` — это команда для профилирования запроса, которая реально выполняет запрос и показывает метрики по каждому шагу плана выполнения.

**На что нужно смотреть:**
1. **Тип сканирования:**
   - `Seq Scan`: Полное сканирование (медленно, если таблица большая).
   - `Index Scan` / `Index Only Scan`: Чтение по индексу (быстро).
2. **Rows (estimate / actual):** Совпадает ли предполагаемое количество строк (`rows=...`) с реальным (`actual rows=...`). Большая разница говорит об устаревшей статистике планировщика.
3. **Actual time:** Показывает реальное время (в мс), затраченное на этот шаг. Самый важный параметр для поиска узких мест (bottlenecks).
4. **Типы соединений:** Какие физические механизмы используются для `JOIN` (например, `Nested Loop`, `Hash Join` или `Merge Join`).
