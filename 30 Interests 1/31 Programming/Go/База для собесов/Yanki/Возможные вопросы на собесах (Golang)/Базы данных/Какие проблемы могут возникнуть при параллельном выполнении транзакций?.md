Какие проблемы могут возникнуть при параллельном выполнении транзакций?

---

Если транзакции идут одновременно, возможны:

1. **Dirty Read — «грязное чтение»**
Одна транзакция читает то, что другая ещё не зафиксировала (комит ещё не произошёл). Если первая транзакция откатится, мы прочитали невалидные данные.
2. **Non-Repeatable Read — «неповторяемое чтение»**
Прочитал строку → кто-то её изменил и зафиксировал → читаешь снова ту же строку → уже другое значение.
3. **Phantom Read — «фантомные строки»**
Прочитал список строк по условию → кто-то добавил новую строку, подходящую под условие → перечитываешь список по тому же условию → появилась лишняя строка.
4. **Lost Update — "потерянное обновление"**
Два процесса одновременно читают одну строку, изменяют её и записывают обратно. Второй перезаписывает изменения первого.
