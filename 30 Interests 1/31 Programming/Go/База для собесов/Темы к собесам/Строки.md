# Строки в Go

**String** — это неизменяемая (immutable) последовательность байтов. В Go строки по умолчанию кодируются в **UTF-8**, а сама переменная строки является **ссылочным типом** (структура заголовка, указывающая на область памяти).

## Базовая информация

- **Неизменяемость (Immutability)**: После создания строку нельзя изменить. Любая модификация (конкатенация, замена символа) создает новую строку в памяти.
- **Нулевое значение**: Пустая строка `""`. Массива байтов за ней в этот момент нет.
- **Безопасность**: Благодаря неизменяемости строки потокобезопасны — их можно передавать между [[30 Interests/31 Programming/Go/База для собесов/Темы к собесам/Горутина|горутинами]] без блокировок.
- **Кодировка**: Строка — это набор байтов, а не символов. Для работы с Unicode (многобайтными символами) используются **руны**.

---

## Внутреннее устройство

В рантайме строка представлена структурой `stringStruct`, которая занимает в памяти **16 байт** (на 64-битном процессоре):

```go
type stringStruct struct {
    str unsafe.Pointer // Указатель на массив байтов (data)
    len int            // Длина строки в БАЙТАХ
}
```

- **Data**: Указывает на область памяти, где лежат байты. Эта область защищена от записи.
- **Len**: Поле длины позволяет функции `len()` работать за $O(1)$. 

> [!warning]
> `len(s)` возвращает количество **байтов**, а не визуальных символов. Для строки `"Привет"` `len` вернет 12, так как каждый символ кириллицы занимает 2 байта.

## Байты, Руны и Юникод

### Rune (Руна)
**Rune** — это псевдоним (alias) для типа `int32`. Она представляет собой один кодовый пункт (Code Point) в стандарте Unicode.

### Итерация по строке
Способ перебора сильно влияет на результат:

```go
s := "Hi 🚢"

// 1. Побайтовый перебор (s[i] возвращает uint8)
for i := 0; i < len(s); i++ {
    fmt.Printf("%x ", s[i]) // Выведет HEX байтов
}

// 2. Перебор по рунам (r возвращает int32/rune)
for idx, r := range s {
    fmt.Printf("%d: %c ", idx, r) // Правильно выведет символы
}
```
**Важно**: В цикле `range` индекс `idx` — это стартовая позиция байта данной руны. Если руна занимает 4 байта, следующий индекс будет `idx + 4`.

## Оптитимизация и Производительность

### Конкатенация
- **Оператор `+`**: Подходит для соединения нескольких строк за один раз. Однако в цикле это крайне неэффективно ($O(n^2)$), так как при каждой итерации выделяется новая память.
- **strings.Builder**: Рекомендуемый способ для сборки строк в цикле.
  ```go
  var sb strings.Builder
  sb.Grow(100) // Оптимизация: заранее выделяем память
  for i := 0; i < 100; i++ {
      sb.WriteString("data")
  }
  result := sb.String()
  ```

### Преобразования `[]byte` <-> `string`
Преобразование `string([]byte)` или `[]byte(string)` почти всегда вызывает **копирование данных**.
- Исключение: В некоторых случаях (например, ключи в `map[string]`) компилятор может оптимизировать копирование, если видит, что слайс байт не меняется.

## Срезы и Утечки памяти

Срез строки `s[1:5]` — это создание новой структуры `stringStruct`, которая указывает на **тот же самый** массив байтов в памяти.

### Опасность Memory Leak
Если вы создаете маленькую подстроку из огромного лога (например, 1 ГБ) и храните её, то весь гигабайт будет висеть в памяти, так как подстрока удерживает ссылку на весь базовый массив.

**Решение**:
```go
// Сделать копию, чтобы отпустить старый массив
newStr := strings.Clone(largeLog[:10]) 
// Или через конвертацию: string([]byte(sub))
```

## Сравнение строк
Строки сравниваются **лексикографически** (побайтово).
- Операторы: ` == `, `!=`, `<`, `>`, `<=`, `>=`.
- Сравнение эффективно, так как рантайм сначала проверяет `len`. Если длины разные, строки точно не равны.

## Приложение: Полезные функции
- `utf8.RuneCountInString(s)` — честный подсчет количества символов ($O(n)$).
- `strings.TrimRight` — удаляет **набор** символов (будьте осторожны!).
- `strings.TrimSuffix` — удаляет конкретный **суффикс**.
