## Что такое Docker?

**Docker** — это программная платформа для быстрой разработки, тестирования и развертывания приложений в контейнерах.

- Docker позволяет запускать **один контейнер** на одном хосте.
- **[[obsidian/База для собесов Копия/Темы к собесам/Kubernetes (k8s)|Kubernetes]]** помогает настраивать сетевую связность между Docker-контейнерами и оркестровать их работу.

## Docker-контейнеры

Docker **упаковывает** программное обеспечение в стандартизованные блоки, называемые **контейнерами**. Каждый контейнер включает **все необходимое** для работы приложения:

- Библиотеки
- Системные инструменты
- Код приложения
- Среду исполнения

### Разница между контейнерами и виртуальными машинами

| Характеристика         | Контейнер                         | Виртуальная машина             |
| ---------------------- | --------------------------------- | ------------------------------ |
| Виртуализация          | Только на уровне ОС               | Виртуализация всего компьютера |
| Использование ресурсов | Меньше ресурсов, быстрее стартует | Требует больше ресурсов        |
| Изоляция               | Ограниченная изоляция             | Полная изоляция                |
| Запуск ОС              | Использует ядро хостовой ОС       | Может запускать любую ОС       |

## Stateful vs Stateless приложения

|Тип|Описание|Применение|
|---|---|---|
|**Stateful**|Приложение **сохраняет состояние** (например, сессии пользователей).|Базы данных, платежные системы|
|**Stateless**|Каждый запрос **независим**, приложение не хранит состояние.|Веб-серверы, REST API|

## Основные команды Docker

### **Команды работы с контейнерами**

| Команда                               | Описание                                        |
| ------------------------------------- | ----------------------------------------------- |
| `docker run <image>`                  | Запуск контейнера из образа                     |
| `docker ps`                           | Список работающих контейнеров                   |
| `docker ps -a`                        | Список всех контейнеров (включая остановленные) |
| `docker stop <container_id>`          | Остановка контейнера                            |
| `docker rm <container_id>`            | Удаление контейнера                             |
| `docker logs <container_id>`          | Просмотр логов контейнера                       |
| `docker exec -it <container_id> bash` | Запуск командной строки внутри контейнера       |

### **Команды работы с образами**

|Команда|Описание|
|---|---|
|`docker images`|Просмотр локальных образов|
|`docker pull <image>`|Загрузка образа из Docker Hub|
|`docker build -t <image_name> .`|Сборка образа из Dockerfile|
|`docker rmi <image_id>`|Удаление образа|

## Dockerfile: Инструкция сборки контейнера

Dockerfile — это файл, содержащий инструкции по созданию Docker-образа.

### Основные команды Dockerfile

| Команда      | Описание                                                      |
| ------------ | ------------------------------------------------------------- |
| `FROM`       | Указывает базовый образ                                       |
| `LABEL`      | Описывает метаданные (например, автора)                       |
| `ENV`        | Устанавливает переменные окружения                            |
| `RUN`        | Запускает команду в процессе сборки образа                    |
| `COPY`       | Копирует файлы в контейнер                                    |
| `ADD`        | Копирует файлы (поддерживает tar-архивы)                      |
| `WORKDIR`    | Устанавливает рабочую директорию в контейнере                 |
| `CMD`        | Указывает команду, которая выполнится при запуске контейнера  |
| `ENTRYPOINT` | Определяет исполняемую команду в контейнере                   |
| `EXPOSE`     | Открывает порты контейнера                                    |
| `VOLUME`     | Создает точку монтирования для работы с постоянным хранилищем |

## Заключение

Docker — мощный инструмент для разработки и развертывания контейнеризированных приложений. Он позволяет:

- Быстро развертывать программное обеспечение.
- Гибко управлять ресурсами.
- Обеспечивать изоляцию и переносимость окружения.

Знание Docker-команд и принципов его работы значительно упрощает процесс DevOps и CI/CD.
