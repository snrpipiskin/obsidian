## **Монолитная архитектура**

Монолит — это серверное приложение с **единой кодовой базой** и точкой запуска.

### **Проблемы монолита**

- **Жесткое связывание** — сложно изменять отдельные модули.
    
- **Сложность тестирования** — изменения требуют регрессионного тестирования всей системы.
    
- **Надежность** — сбой в одном месте может привести к отказу всей системы.
    
- **Деплой** — разворачивается целиком, сложно обновлять отдельные компоненты.
    
- **Масштабирование** — приходится масштабировать всю систему, даже если нагрузка растет в одном из модулей.

## **Микросервисная архитектура**

Микросервисы — это разделение системы на **изолированные сервисы**, каждый из которых выполняет **одну бизнес-задачу**.

![[Untitled 9.png]]

### **Преимущества микросервисов**

- **Гибкость** — можно развивать и обновлять отдельные сервисы без влияния на другие.
    
- **Лучшая отказоустойчивость** — выход из строя одного сервиса не ломает всю систему.
    
- **Масштабируемость** — можно масштабировать только те сервисы, которые испытывают нагрузку.
    
- **Независимый деплой** — команды могут разрабатывать и развертывать сервисы отдельно.
    

### **Проблемы микросервисов**

- **Сложность** — усложняется инфраструктура и управление.
    
- **Коммуникация** — передача данных между сервисами через сеть приводит к задержкам и потенциальным сбоям.
    
- **Консистентность данных** — нужна продуманная стратегия согласованности.
    
- **Безопасность** — увеличивается количество точек взаимодействия, требующих защиты.

## **API Gateway**

API Gateway ставят перед сервисами, чтобы пользователь **не общался с ними напрямую**.

### **Функции API Gateway**

- **Маршрутизация** — передает запросы нужным микросервисам.
    
- **Агрегация** — собирает данные из нескольких сервисов в один ответ.
    
- **Безопасность** — контроль доступа, аутентификация и авторизация.
    
- **Кеширование** — снижение нагрузки на сервисы за счет хранения часто запрашиваемых данных.

## **Коммуникация между микросервисами**

### **Синхронная коммуникация**

- [[obsidian/База для собесов Копия/Темы к собесам/Сети|HTTP]] — REST API, стандартный протокол передачи данных.
    
- **gRPC** — бинарный протокол, более быстрый и эффективный, чем HTTP.
    

### **Асинхронная коммуникация**

- **Kafka** — распределенная очередь сообщений, используемая для обработки событий в реальном времени.
    
- **RabbitMQ** — брокер сообщений для обмена данными между сервисами.
