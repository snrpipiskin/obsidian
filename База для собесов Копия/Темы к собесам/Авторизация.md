## ==**База по авторизации**==

**Авторизация** — проверка прав доступа пользователя в системе.

Информация зашивается в токен, поэтому база данных для авторизации не требуется.

---

## **Аутентификация vs Авторизация**

- **Аутентификация** — способ определить, кто перед нами.
    
- **Авторизация** — проверка, имеет ли пользователь право на доступ к определенным ресурсам.
    

---

## **Процесс авторизации**

1. Пользователь делает запрос к [[obsidian/База для собесов Копия/Темы к собесам/Сети|URL]].
    
2. Необходимо пройти процесс аутентификации.
    
3. Система авторизации выдает пользователю токен (например, JWT).
    
4. При повторных запросах пользователь прикладывает этот токен для проверки доступа.
    

---

## **JWT (JSON Web Token)**

**JWT** — открытый формат передачи токенов, который используется для авторизации.

- Позволяет передавать зашифрованную информацию между клиентом и сервером.
    
- Состоит из трех частей: заголовок (header), полезная нагрузка (payload) и подпись (signature).
    
- Позволяет системе работать без хранения сессий на сервере, поскольку вся необходимая информация уже содержится в токене.
    

Пример JWT токена:

```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
  eyJ1c2VySWQiOiIxMjM0NTY3ODkwIiwicm9sZSI6ImFkbWluIn0.
  SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
```
